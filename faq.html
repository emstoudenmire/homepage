Frequently Asked Questions

What are tensor networks?

Tensor networks are a way of representing a very large tensor (a multi-dimensional array of numbers) as a contracted network of smaller tensors. By choosing the right network geometry, you can capture the most important structure of the data while using exponentially fewer parameters than a brute-force representation. They originated in quantum physics but are finding applications in machine learning, signal processing, and beyond.

What is DMRG?

DMRG stands for the Density Matrix Renormalization Group. It is a numerical algorithm for finding the ground state (lowest-energy state) of quantum many-body systems, especially one-dimensional systems. DMRG works by iteratively optimizing a matrix product state (MPS) representation of the quantum wavefunction, and it is widely considered the gold-standard method for 1D quantum lattice problems.

What is ITensor?

ITensor is an open-source software library for tensor network calculations. It provides a high-level interface where tensors have named indices, inspired by tensor diagram notation. This makes it much easier to write and debug tensor network code compared to working with raw multidimensional arrays. ITensor is available in both Julia and C++.

How can tensor networks be used in machine learning?

Tensor networks can serve as machine learning models that are mathematically well-understood and have built-in structure inspired by quantum physics. For example, a matrix product state can be used as a supervised classifier by mapping input data into a high-dimensional feature space and contracting the network to produce a prediction. These models offer advantages in interpretability and can be trained using techniques borrowed from DMRG.

What programming language should I use for tensor networks?

For most new projects, I recommend using the Julia version of ITensor (ITensors.jl). Julia combines the ease of use of a high-level language like Python with performance comparable to C or Fortran. The Julia version of ITensor is also the most actively developed. If you need to interface with an existing C++ codebase, the C++ version of ITensor is also available and well-tested.
